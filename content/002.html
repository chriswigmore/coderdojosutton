Introduction

Motion

Motion

Fireball

Fireball

Click on the Costumes tab.

You should see three different dragon costumes. Click the last one, where the dragon is breathing fire.

Click to select the fireball and use Ctrl + X to cut it to your clipboard.

Now we need to create a new sprite, but don't choose one from the library, instead select 'Paint'.

Use Ctrl + V to paste in the fireball. You will probably see a mess of blue circles and outlines and arrows — press the button to zoom in until you see fire.

The fireball is positoned high and over on the right — drag it until it's in the middle of the drawing area.

It's also angled slightly downwards — use the curved blue arrows to turn the fireball until it's pointing straight to the right.

Rename this sprite 'Fireball'.

How do I rename a sprite?

Type a new name in the Sprite's name box, and hit Enter.

Next, we'll let the dragon blast out lots of fireballs using 'clones'.

Clones

Clones

Change from the Costumes tab back to the Code tab. Add to the Dragon sprite a create clone of myself (from the 'Control' section) and change it to create clone of Fireball. Click it once. What happens?

Wait - there's no option to create clone of Fireball?

You probably added the block to the Fireball sprite — make sure you add the block to the Dragon sprite!

Drag the fireball to somewhere else on the screen. What do you see?

When you created a clone of the fireball, the clone was identical to the first fireball — the template fireball as we'll call it. It looked the same, it even had the same position and direction as its template, which is why you couldn't see it until you moved the template out of the way.

Click to create another clone (you'll need to switch back to the Dragon sprite) and drag the template again — you should now see three fireballs: one template, two clones.

Click the stop sign - what happens?

If we want to run some code when a clone is first created, we can snap that code underneath... what? Can you guess?

Add to the Fireball sprite a when I start as a clone (from the 'Control' section).

We want to make the clone fly across the screen, so we want to wrap some kind of motion block inside some kind of loop. However, let's get into the good habit of thinking about a clone's lifetime. There is a maximum number of clones you can create, so if we keep creating them and never deleting them, what will happen?

Any clones that we no longer need should be deleted, and once a fireball clone has flown across the screen we no longer need it. So, snap on a repeat until (from the 'Control' section), put a touching edge in its slot, nest a move 10 steps block inside the loop, and snap a delete this clone underneath.

Positioning

Positioning

In the fireball's code area, add a go to random position, change it to go to Dragon and click it.

The fireball is probably over the dragon's belly now. Don't drag the sprite itself to position it, we need to add an 'offset' into the costume itself — here's how:

Click on Costumes, use Ctrl + A to select the fireball, and drag it up and to the right a little bit. Let go and look at the stage — you should see that the fireball has moved a little closer to the dragon's mouth. Keep dragging it until it's coming out of the dragon's mouth.

Drag the go to Dragon in between the when I start as a clone and repeat untiltouching edge, because the dragon can move left and right, so we need to position every fireball correctly at the moment it's created.

Try walking left and right and blasting out fireballs. What do you see?

Let's fix the direction bug first. Snap in a point in direction 90 before the repeat loop. In its slot, add a backdrop # of Stage block (from the 'Sensing' section) and change it to direction of Dragon.

This will make the fireball point in the same direction as the dragon, but we need to set its rotation style to left-right, so add a when clicked and snap a set rotation style left-right underneath.

Test the change. What do you see now?

We never need to see the template fireball, only the clones that fly across the screen, so snap a hide block (from the 'Looks' section) underneath the when clicked script. Test the change. What do you see now?

Remember that a clone is created identical to its template. We hid the template, so the clones were hidden too. If we want to hide the template but show the clones, we need to add a show when each clone is created, right before the repeat loop.

Test the change, shooting left and right. What do you see now?

Good. There's one last bug to fix. If you press and hold both an arrow key and the space bar, it looks like a fireball is glued to the dragon's mouth again. Why do you think that happens?

Set the fireball clones to move 5 steps at a time. You'll see the dragon overtake its own fireballs.

If you set them to 11, the fireballs will be slightly faster than the dragon. Setting them to 15 looks good to me — feel free to set it to a different number if that looks better to you.

Tip: Fireballs can be too fast!

Remember, the fireball only looks like it's flying smoothly through the air, it's actually jumping from position to position. In a minute we'll add a snake that we need to shoot — if our fireballs travel too fast they will 'jump over' the snake without hitting it.

Snake

Snake

Add a snake sprite. Change its size to 60.

Start it off on the right hand side by adding a when clicked with a go to x: 280 y: -60 underneath.

Point the snake to the left (direction -90) and set its rotation style to left-right.

We want the snake to move towards the dragon forever, so snap on a forever loop with a move 3 steps nested inside (10 steps feels too fast).

If the snake gets hit by a fireball, it should be knocked back to its starting position at the right hand side. Can you work out what blocks to add inside the forever loop?

If the snake manages to bite the dragon, it should be game over. Can you work out what blocks to add inside the forever loop?
