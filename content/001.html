Sensing and Control

Sensing

Add a key space pressed? block (from the 'Sensing' section) to the code area. Change it to key up arrow pressed?. What happens when you click on it?

Press and hold the up arrow key, and click the block again. What happens now?

Whenever this block 'runs' (the easiest way to run a block is to click on it) it returns true if the up arrow is pressed, false if it's not. (It senses whether the up arrow is pressed.)

It is an example of a 'Boolean', or 'bool' for short. A bool is a thing which is either true or false. The up arrow key can't be half-pressed; it's either pressed or it's not.

Add an if then block (from the 'Control' section).

What do you notice about the shape of the if then block's slot, and the shape of the key up arrow pressed? block?

You see, if then blocks decide if they should run the code nested inside them. They will either run it or not; you can't half-run some code. Therefore, they work with bools, so it makes sense that their slot is the same hexagonal shape as the bool blocks.
